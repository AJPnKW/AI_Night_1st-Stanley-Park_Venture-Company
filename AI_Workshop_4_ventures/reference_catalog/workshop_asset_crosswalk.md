# Workshop Asset Crosswalk

## Purpose

Map the existing workshop-pack files to the current workshop-system needs, identify what each file already covers, what is outdated, what gaps remain, and what child-root docs should reference each source.

## Crosswalk

| Existing File | What It Already Covers | What Is Outdated | Gaps Remaining | New Detailed Docs That Should Reference It | Recommendation |
| --- | --- | --- | --- | --- | --- |
| `workshop_pack\learning_plan_ai_chatgpt_session.html` | 90-minute structure, learning goals, explain-demo-do-reflect cadence, neurodiversity-supportive intent | still framed as a general AI engines/tools/agents session rather than the current ChatGPT-centered workshop system | tighter one-hour fallback, explicit parent/pre-work linkage, facilitator adaptation cues | `learning_plan_expansion.md`, `exercises_and_activity_bank.md`, `workshop_resource_expansion_audit.md` | keep as historical/source spine and maintain a child-root companion file |

## Summary

- None of the pack files should be treated as final truth.
- All five remain useful source assets.
- The child-root reference-catalog docs are now the authoritative detailed guidance.
- The pack should stay preserved, with companion docs doing the current alignment work.
