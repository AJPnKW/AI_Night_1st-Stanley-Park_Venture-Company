# What Beginners Are Not Told

## Purpose

Capture the missing truths that often get skipped in first AI sessions so learners do not confuse polished output with trustworthy output.

## The Gaps

### 1. AI Can Sound Sure And Still Be Wrong

- a polished answer is not proof
- important facts, dates, links, and claims still need checking

### 2. The First Answer Is Usually A Draft

- better prompting usually means better results
- iteration is normal, not failure

### 3. Bias And Gaps Still Show Up

- outputs may reflect uneven training patterns
- learners should compare perspectives and notice what is missing

### 4. Privacy Is Still The User's Responsibility

- the tool does not know what should stay private
- learners must avoid sensitive or identifying details

### 5. Convenience Can Create Overtrust

- speed feels useful, which can reduce careful checking
- the right habit is “use AI to support thinking, not replace it”

## Useful Habit Pairings

- wrong-but-confident -> verify one key claim
- weak prompt -> add goal, audience, and format
- privacy risk -> remove names and specifics
- biased answer -> ask for another perspective
- overtrust -> compare with another source
