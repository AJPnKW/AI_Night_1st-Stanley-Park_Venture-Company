Participant Workbook Gap Assessment

Gap assessment between the current local workbook and the richer participant-owned workbook model requested for the event.

What the request really means

The workbook request is not just about adding more fields to a form. It describes a participant-owned learning guide that combines instruction, note capture, exercise choice, adaptive follow-up, private retention, and extension work for faster learners.

Current state versus desired state

Privacy and retentionLocal browser save, download, and print exist.Each learner should feel the workbook belongs to them and keeps their notes private unless they choose otherwise.Mostly met in the browser-only path.
Exercise guidanceExercise cards exist and the workbook page can capture notes.Workbook should feel like a guided booklet with clearer flow and fewer disconnected steps.Partly met.
Adaptive coachingThe local workbook can now show helper prompts based on category, level, and result.Follow-up should change based on what the learner says happened and what gap remained.Partly met in the client-side path.
Tracked learner workflowNo centralized tracking on GitHub Pages.Organizer should be able to issue one workflow per participant and retain submissions centrally if desired.Blocked on a supplemental Forms layer.
Fast-finisher pathExtension guidance exists in the local workbook.Fast learners should be routed into meaningful follow-up work instead of waiting.Mostly met locally; stronger with Forms branching.
Leader/operator visibilityNo central workbook view in the static Pages-only path.Organizer should be able to see who completed what and who needs help.Requires Google Sheets or equivalent.

Practical conclusion

The current GitHub Pages workbook is a good private workbook fallback and live-session notebook. The full participant-owned LMS-like experience requires a forms-backed workflow layer for participant routing, structured response capture, and operator visibility.
