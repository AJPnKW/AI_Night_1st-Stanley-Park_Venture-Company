Participant Workbook Sheet Schema

Recommended Google Sheets structure for participant workbook response capture and operational visibility.

Workbook response schema

ColumnPurpose
timestampWhen the response was submitted
participant_idName, initials, or private label
group_namePatrol, team, or table if used
roleParticipant, leader test, backup, or other route
device_routeBrowser-first, fallback, or optional advanced path
comfort_levelSelf-reported comfort or support need
categoryChosen exercise category
levelGuided, Build, Remix, or Stretch
starter_promptSelected weak starter prompt
improved_promptParticipant improved prompt
what_changedPrompt-improvement notes
expected_resultWhat the learner hoped to get
result_ratingYes mostly, partly, or not yet
output_qualityUseful already, getting closer, still weak, or not sure yet
gap_remainingWhat was still missing
verify_nextWhat still needs checking
next_tryPlanned next revision
extension_usedFast-finisher or optional extra work
needs_helpLeader/facilitator flag
follow_up_statusNot reviewed, reviewed, follow-up needed, closed

Suggested tabs

  • Responses raw
  • Leader view
  • Completion dashboard
  • Participant link registry
  • Prompt card reference
  • Post-event follow-up

Operational uses

  • See who has not started or has stalled.
  • See which learners flagged support needs.
  • See which categories and levels were most used.
  • See who finished early and used extension work.
  • Filter for follow-up or accommodation notes after the session.
