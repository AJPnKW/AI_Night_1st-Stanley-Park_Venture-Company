Participant Workbook Apps Script Spec

Optional Apps Script design for email, summary, PDF, and participant-link automation around the workbook flow.

Why this is optional

The event can run successfully without Apps Script. This layer only becomes useful if the organizer wants centrally stored workbook output, summary emails, or automated completion handling.

Suggested script functions

FunctionPurpose
createPrefilledLinks()Create one prefilled form URL per participant and write it to the participant link registry sheet.
onFormSubmit(e)Read the submission, calculate a simple completion status, and flag help-needed responses.
buildParticipantSummary(row)Format a plain-language learner summary from the response row.
emailParticipantSummary(row)Send the learner or family a summary email if the organizer enables that path.
exportPdfSummary(row)Generate a PDF copy of the learner workbook summary and store or email it.
updateDashboard()Refresh completion counts and support flags for the leader dashboard.

Workflow outline

StepAction
1. Before the eventGenerate participant links from the roster sheet.
2. During the eventLearners use their issued workbook link.
3. On submitApps Script writes status fields and optional support flags.
4. After the eventOrganizer can email summaries or export PDFs if enabled.
5. Follow-upDashboard shows who completed, who needs follow-up, and what topics were most used.

Guardrails

  • Do not promise this layer unless the organizer actually configures it.
  • Keep it separate from the public GitHub Pages site.
  • Treat any email or personal data flow as organizer-owned operational configuration, not public website behavior.
